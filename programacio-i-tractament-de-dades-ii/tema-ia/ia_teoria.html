Etapa 1 — Codi escrit completament a mà

Anys 90 fins a principis dels 2000. El programador escrivia cada línia sense cap ajuda intel·ligent.

⌨️ Com es programava

Els programadors escrivien tot el codi des de zero, usant editors de text simples com el Bloc de Notes, Notepad++ o terminals de text. No hi havia suggeriments, no hi havia detecció d'errors en temps real — si t'equivocaves, ho descobries quan el programa fallava.

🛠️ Eines típiques de l'època

  • Notepad, vim, emacs
  • Compiladors de línia de comandes
  • Debuggers bàsics de terminal
  • Manuals en paper o PDF

🧠 Habilitats necessàries

  • Memoritzar sintaxi al detall
  • Conèixer bé l'API de cada llibreria
  • Depurar mentalment el codi
  • Llegir documentació extensa
⏱️ Una tasca senzilla com llegir un fitxer CSV i calcular la mitjana d'una columna podia costar 30-60 minuts, incloent la consulta de documentació i la depuració d'errors.

🖥️ Entendre el maquinari i el programari

En aquella època, un bon programador havia d'entendre com funcionava la màquina per dins. La relació entre hardware i software era molt més directa.

🔩 Hardware

  • Components físics de l'ordinador
  • Processador, memòria RAM, disc dur
  • Perifèrics: teclat, ratolí, monitor

💾 Software

  • Sistema: SO (Windows, Linux)
  • Eines: Compiladors, editors
  • Aplicació: Programes d'usuari

🔄 Compilació vs Interpretació

Ja en aquesta etapa existia la diferència clau entre compilar (traduir tot el codi a binari d'una vegada) i interpretar (executar-lo línia a línia en temps real, com fa Python).

⚙️ Compilats (C, C++)

  • Tradueixen tot el codi a màquina
  • Generen un executable (.exe)
  • Molt ràpids d'executar
  • Fitxers diferents per SO

🐍 Interpretats (Python)

  • Llegeixen i executen en temps real
  • Mateix codi per a tots els SO
  • Cal l'intèrpret instal·lat
  • Més fàcils de provar i aprendre
💡 Python, creat el 1991, va néixer com a resposta a la complexitat dels compilats: un llenguatge llegible, interpretat i fàcil d'aprendre. Aquesta filosofia el fa perfecte per aprendre a programar.

📐 Algorismes i pseudocodi: pensar abans de codificar

Sense eines d'ajuda, el disseny previ era fonamental. Un programador havia de tenir molt clar el que volia fer abans d'escriure la primera línia de codi.

Un algorisme és una seqüència de passos organitzats per resoldre un problema, independent del llenguatge. El pseudocodi és una forma d'escriure'l en un llenguatge semblant a l'humà.

// Pseudocodi: trobar el nombre màxim d'una llista
INICI
  llegir llista de nombres
  maxim ← primer element de la llista
  PER CADA nombre DE la llista:
    SI nombre > maxim:
      maxim ← nombre
  mostrar maxim
FI
🎯 Aquesta habilitat segueix sent fonamental avui, fins i tot amb IA. Si no saps plantejar un problema pas a pas, tampoc podràs dir-li a una IA el que vols que faci.

Etapa 2 — Autocompleció bàsica

2005–2015. Els IDEs comencen a suggerir noms de funcions i variables mentre el programador escriu.

🔮 Què és un IDE i per què va canviar-ho tot

Un IDE (Entorn de Desenvolupament Integrat) és una aplicació que reuneix en un sol lloc tot el que necessita un programador: editor de text, intèrpret/compilador i depurador.

A diferència d'un simple editor de text, l'IDE entén el codi: sap quines funcions has definit, quines variables existeixen i quines llibreries has importat.

📦 Exemples d'IDEs populars

  • VS Code — el més usat avui
  • PyCharm — especialitzat en Python
  • Eclipse — Java i altres
  • Xcode — Apple (iOS/macOS)

🧩 Què té un IDE

  • Editor de text amb colors de sintaxi
  • Compilador o intèrpret integrat
  • Depurador visual (debugger)
  • Terminal integrat
  • Gestió de fitxers del projecte

💡 Autocompleció: la primera ajuda intel·ligent

La primera gran millora va ser l'autocompleció: mentre escrius, l'IDE t'ofereix una llista de suggeriments basats en el codi que has escrit fins ara — noms de funcions, variables, mètodes de classes importades.

# Quan escrius "fr" i prems Tab, l'IDE suggereix:
from math import sqrt   # ← autocompleta "from"

fruites = ["poma", "pera"]
fruites.ap  # ← l'IDE suggereix: .append(), .apply()...
📈 L'autocompleció reduïa els errors tipogràfics i accelerava l'escriptura, però encara era bàsica: no entenia el context semàntic ni sabia si el codi tenia sentit lògic.

🎨 Ressaltat de sintaxi i navegació

Dues funcions senzilles però impactants: els IDEs van aprendre a colorar el codi per tipus d'element (paraules clau en blau, strings en vermell, comentaris en gris...) i a permetre navegar saltant directament a la definició d'una funció.

🔍 Avui sembla trivial, però el ressaltat de sintaxi va reduir dràsticament els errors: és molt més difícil confondre una variable amb una paraula reservada si tenen colors completament diferents.

🚀 Extensions i ecosistema

Els IDEs moderns com VS Code van portar aquest concepte molt més lluny: un sistema d'extensions que permet afegir funcionalitats infinites. Per a Python, la extensió oficial de Microsoft activa totes les funcions avançades.

✅ Per programar Python en VS Code necessites: Python 3 instal·lat + l'extensió oficial de Python (Microsoft) + executar python --version al terminal per verificar que tot funciona.

Etapa 3 — IntelliSense i Linting

2015–2020. Les eines aprenen a entendre el codi de forma profunda: suggeriments intel·ligents, detecció d'errors en temps real i anàlisi de qualitat.

🧠 IntelliSense: molt més que autocompleció

IntelliSense és el nom que Microsoft va donar al seu motor d'assistència intel·ligent dins VS Code. Va molt més enllà de suggerir noms: entén el tipus de cada variable i el que cada funció espera rebre.

FuncióQuè fa exactament
Autocompleció intel·ligentSuggereix mètodes vàlids per al tipus de dada concret que estem usant
Info de paràmetresMostra quins arguments accepta una funció i de quin tipus han de ser
Hover documentationPassa el cursor per sobre d'una funció i veus la documentació completa
Go to definitionClic + F12 per saltar directament a on es defineix una funció o classe
Find all referencesTroba tots els llocs del codi on s'usa una variable o funció
 
# IntelliSense sap que "fruites" és una llista i suggereix:
fruites = ["poma", "pera", "kiwi"]
fruites.  # → mostra: append, remove, sort, reverse, index, count...

# I si poses el cursor sobre sorted(), mostra la seva documentació:
resultat = sorted(fruites)  # → sorted(iterable, key=None, reverse=False)

🔍 Linting: el revisor automàtic

El linting és l'anàlisi automàtica del codi per trobar errors, males pràctiques i problemes d'estil sense necessitat d'executar-lo. Com un corrector ortogràfic però per a codi.

📌 El nom ve de "lint" — la pelussa que s'acumula a la roba. Petites impureses al codi que cal eliminar per tenir un projecte net.
# El linter marcaria tot això com a errors o advertències:
def calcular( x,y ):       # ⚠️ Espais incorrectes als parèntesis
    z=x+y                   # ⚠️ Falta espai al voltant de =
    return z
    print("mai s'executa") # ⚠️ Codi mort (unreachable code)

variable_no_usada = 42    # ⚠️ Variable definida però mai usada

🐍 Linters per a Python

  • Pylint — el més complet
  • Flake8 — ràpid i lleuger
  • Ruff — modern, molt ràpid
  • mypy — comprovació de tipus

📏 PEP 8: l'estàndard d'estil

Guia oficial d'estil de Python. Defineix espaiat, longitud de línies, nomenclatura, imports... Els linters comproven que el teu codi el compleixi automàticament.

📊 Impacte real del linting als equips

En empreses i equips de programadors, el linting va passar a ser obligatori: el codi no es podia pujar al repositori si el linter trobava errors. Això va millorar dràsticament la qualitat i la llegibilitat dels projectes.

Sense linting

  • Cada programador té el seu estil
  • Els errors es descobreixen executant
  • Codi difícil de llegir en equip
  • Bugs subtils passen desapercebuts

Amb linting

  • Estil unificat a tot el projecte
  • Errors detectats mentre escrius
  • Codi consistent i llegible
  • Males pràctiques eliminades
✅ La combinació perfecta d'aquesta etapa: IntelliSense t'ajuda a escriure codi correcte, Linting t'assegura que sigui de qualitat. Tots dos actuen en temps real mentre programes.

Etapa 4 — IA Generativa i Copilots de codi

2021–2023. La IA deixa de revisar el codi i comença a generar-lo. Un salt qualitatiu enorme.

✨ Què és la IA Generativa?

La IA Generativa és el tipus de IA capaç de crear contingut nou que no existia abans: text, codi, imatges, vídeos, música... No es limita a classificar o revisar — genera.

La base de la IA generativa de text i codi és el Large Language Model (LLM): un model entrenat amb quantitats massives de text (i codi) per aprendre com funciona el llenguatge i poder continuar qualsevol seqüència de text de forma coherent.

🧠 Un LLM no "busca" la resposta en una base de dades. Genera text token a token, predient cada vegada quina paraula és més probable que segueixi el context anterior.

⚙️ Com funciona un LLM (simplificat)

1

Tokenització

El text es divideix en tokens (paraules o fragments). "Programació" pot ser 1 token; "anticonstitucional" pot ser-ne 4.

2

Embeddings

Cada token es converteix en un vector numèric. Tokens amb significat similar queden propers en aquest espai matemàtic.

3

Mecanisme d'Atenció (Attention)

El model decideix quins tokens anteriors són més rellevants per predir el següent. Pot "atendre" paraules molt llunyanes al text.

4

Generació token a token

El model genera un token cada vegada. Cada nou token s'afegeix al context i influencia el següent fins completar la resposta.

🐙 GitHub Copilot: el primer copilot massiu

Llançat el 2021 per GitHub (Microsoft) en col·laboració amb OpenAI. Basat en el model Codex (derivat de GPT-3) entrenat amb milers de milions de línies de codi públic de GitHub.

Funciona com una extensió de VS Code. Mentre escrius, analitza el context — el fitxer obert, els comentaris, el nom de les funcions — i suggereix blocs de codi complets en temps real.

# Amb un simple comentari, Copilot genera la funció sencera:
# Funció que comprova si un nombre és primer
def es_primer(n):          # ← Copilot autocompleta tot el de baix
    if n < 2: return False
    for i in range(2, int(n**0.5)+1):
        if n % i == 0: return False
    return True

🌀 Cursor: l'IDE redissenyat per a la IA

Cursor és un IDE basat en VS Code però dissenyat des de zero per integrar IA en cada pas del treball. No és una extensió afegida — la IA és el nucli del producte.

Característiques clau

  • Xat directament al codi
  • Edita múltiples fitxers alhora
  • Entén tot el projecte sencer
  • Usa Claude, GPT-4 o Gemini

Per què és diferent

Pots dir-li: "Refactoritza aquesta funció i actualitza tots els tests que la criden". I ho fa en tots els fitxers alhora, entenent tot el context del projecte.

💬 La gran novetat de Cursor: pots xatejar amb la IA sobre el teu codi. "Per què falla aquesta funció?", "Com puc fer-la més eficient?", "Afegeix gestió d'errors". La IA respon en context.

💬 Prompting: com parlar amb la IA

Un prompt és la instrucció que li dones al model. La qualitat del prompt afecta directament la qualitat de la resposta.

❌ Prompt feble

"Fes-me una funció Python"

Massa vague. La IA no sap per a què, amb quines dades ni quin format de sortida vols.

✅ Prompt fort

"Crea una funció Python que rebi una llista d'enters i retorni la mitjana. Inclou docstring i gestió d'errors si la llista és buida."

Etapa 5 — Agents d'IA i el futur de la programació

2024 en endavant. La IA deixa de ser una eina i comença a ser un company de feina que actua de forma autònoma.

🕹️ Agents d'IA: la IA que actua

Un agent d'IA no es limita a respondre — actua. Pot planificar passos, usar eines, executar codi, llegir fitxers, navegar per la web i prendre decisions per aconseguir un objectiu, tot de forma autònoma.

🎯 Diferència clau: una IA de xat parla. Un agent fa coses per aconseguir un objectiu sense que l'hagis de guiar pas a pas.
💭

Planificació

L'agent analitza l'objectiu i el divideix en subtasques: "Per fer X haig de fer A, després B, i finalment C."

🔧

Ús d'eines

Pot cridar funcions externes: cercar al web, executar Python, llegir/escriure fitxers, consultar una API, enviar emails...

🔄

Feedback loop

Executa una acció, observa el resultat i decideix el pas següent. Si hi ha un error, l'intenta solucionar automàticament.

✅

Resultat final

Quan ha completat tots els passos, entrega el resultat: el codi funciona, el document és llest, l'informe és redactat.

Claude és el model creat per Anthropic (fundada el 2021 per antics membres d'OpenAI). S'ha dissenyat posant èmfasi en la seguretat, el raonament i en ser útil, inofensiu i honest.

🚀 Com canvia el rol del programador

Programador clàssic

  • Escriu cada línia de codi
  • Memoritza sintaxi i APIs
  • Depura errors manualment
  • Prototipa lentament

Programador + IA (avui)

  • Dirigeix i revisa la IA
  • Sap quines preguntes fer
  • La IA proposa, el dev decideix
  • Prototipa 10x més ràpid

🔥 Habilitats cada cop més valuoses

  • Pensament crític i resolució de problemes
  • Saber fer bons prompts
  • Revisar i avaluar codi generat
  • Disseny d'arquitectura de sistemes
  • Coneixements de seguretat

⚠️ Riscos a tenir en compte

  • Codi incorrecte amb aparença correcta
  • Dependència excessiva sense aprendre
  • Privacitat del codi enviat al núvol
  • Possibles problemes de copyright
🎯 La conclusió: aprendre a programar segueix sent essencial. Sense entendre les bases, no pots dirigir ni avaluar la IA. El futur és humà + IA, no IA sola.
