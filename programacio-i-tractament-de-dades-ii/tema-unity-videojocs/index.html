Etapa 1 — Conceptes bàsics d'Unity

Entenent el motor, la interfície i els primers passos de desenvolupament

🎮 Què és Unity?

Unity és el motor de desenvolupament de videojocs més popular del món. Permet crear jocs 2D i 3D per a múltiples plataformes (PC, móvil, consola...) amb un sol projecte base.

Unity es basa en un sistema de componentes: cada element del joc (personatge, enemy, moneda...) és un GameObject que conté components com físiques, renderer de gràfics, scripts de comportament, etc.

📊 Números clau d'Unity

  • Fundat el 2004
  • Gratuït per a menys de 100.000 € ingressos anuals
  • Suporta 25+ plataformes
  • Usat per estudi com Pokémon, Genshin Impact

🎯 Avantatges d'Unity

  • Senzill d'aprendre per a principiants
  • Sistema visual potent (drag-and-drop)
  • Scriptures en C#, un llenguatge poderós
  • Multiplataforma amb pocs canvis
📌 La filosofia d'Unity: fer que és fàcil crear un joc simple, però també que és possible crear jocs complexos. Això la fa ideal tant per a principiants com per a studios professionals.

📥 Instal·lació i configuració inicial

Per començar, necessites descarregar Unity Hub, que és el gestor d'instal·lacions d'Unity. Després, triaràs la versió d'Unity que vols (LTS és la més estable) i crearás un projecte nou.

✅ Passos: 1) Descarrega Unity Hub, 2) Crea un compte Unity, 3) Instal·la la versió 2022 LTS o superior, 4) Crea un projecte "2D" o "3D", 5) Espera que Unity descargui totes les dependències.

🖼️ La interfície d'Unity

L'editor d'Unity es divideix en vàries finestres (windows) que pots organitzar. Les principals són:

FinestraPropòsit
Viewport (Centre)Vista 3D/2D del joc mentre edites la escena
Hierarchy (Esquerra)Llista de tots els GameObjects de la escena actual
Inspector (Dreta)Propietats del GameObject seleccionat
Project (Baix)Fitxers del projecte: scripts, imatges, sons, etc.
Scene (Baix esquerra)Vista general de la escena, zoom i viewport control

🔄 El Sistema de Coordenades

Tots els GameObjects a l'escena tenen una posició representada per coordenades X, Y, Z.

  • X: Esquerra (negatiu) o dreta (positiu)
  • Y: Baix (negatiu) o amunt (positiu)
  • Z: Enrera (negatiu) o endavant (positiu, cap a la càmera)
💡 En 2D, només usaràs X i Y. Z es reserva per a la profunditat (ordre de dibuix).

Etapa 2 — Escenes i GameObjects

Construint el teu primer nivell: com organitzar elements i com interactuen

🎬 Què és una Escena?

Una escena és un nivell o pantalla del joc. Cada escena conté tots els GameObjects, ambient, càmeres, etc. que necessites per a aquesta part del joc.

Un projecte típic pot tenir múltiples escenes: "MainMenu.unity", "Level_1.unity", "GameOver.unity", etc.

📌 Consell: Mantén les escenes organitzades en carpetes dins de la carpeta "Assets". Exemple: Assets/Scenes/Level1, Assets/Scenes/MainMenu, etc.

🎲 Què és un GameObject?

Tot allò que veus a la pantalla és un GameObject: el personatge principal, els enemics, les plataformes, les monedes, les lumieres... fins i tot la càmera.

Un GameObject per si sol no fa res. Els seus components són els que li donen funcionalitat: físiques, gràfics, comportament, so, etc.

📐 Transform: Posició, Rotació i Escala

Cada GameObject té un component Transform. Aquest component controla els tres aspectes visuals més bàsics:

PropietatDescripcióUnitats
PositionOn està el GameObject a l'escena (X, Y, Z)Unitats de l'escena
RotationGirar el GameObject als tres eixosGraus (0–360)
ScaleMida relativa (1 = mida normal, 2 = doble)Factor (0.5–5)
🎯 Pots controlar el Transform manualment a l'Inspector, o canviar-lo des d'un script (per fer moure el personatge, girar objectes, etc.)

👶 Parent-Child (Jerarquies)

Els GameObjects poden ser "fills" (children) d'altres GameObjects dentro de la Hierarchy. Esto es útil para grupos lógicos, como todos los elementos de la UI dentro de un Canvas, o las partes de un personaje.

⚙️ Quan moue el pare, els fills es mouen amb ell. Això simplifica molt la gestió de grups d'objectes que sempre han d'estar junts.

Etapa 3 — Scripts i Components

Programando el comportamiento: física, entrada del usuario, lógica del juego

📝 Escribiendo tu Primer Script

Los scripts en Unity se escriben en C#, un lenguaje compilado y fuerte que es muy similar a Java. Cada script es una clase que hereda de MonoBehaviour.

Cuando creas un script en Unity, obtienes automáticamente dos métodos principales:

using UnityEngine;

public class MiPrimerScript : MonoBehaviour
{
    void Start()
    {
        // Se ejecuta UNA VEZ al principio
        Debug.Log("El juego ha comenzado!");
    }

    void Update()
    {
        // Se ejecuta CADA FRAME (~60 veces por segundo)
        Debug.Log("Actualizando...");
    }
}
✅ Start(): Inicialización. Update(): Comportamiento contínuo cada fotograma.

⌨️ Entrada del Usuario (Input)

Para que el jugador controle el personaje, necesitas leer las pulsaciones del teclado o clicks del ratón. En Unity, esto se hace con Input.GetKey() o Input.GetKeyDown().

🔫 Detección de Colisiones (Colliders)

Para que el personaje não atravese muros ni para que detectes cuando toca una moneda o un enemy, necesitas colliders.

Dos tipos principales:

📦 Box Collider 2D

  • Forma de rectángulo
  • Ideal para plataformas y muros

⭕ Circle Collider 2D

  • Forma de círculo
  • Mejor para objetos redondeados
⚠️ Recuerdo: para que dos colliders detecten colisión, al menos uno debe tener un Rigidbody.

🤝 Métodos Callback de Colisión

Cuando dos colliders chocan, Unity ejecuta automáticamente estos métodos:

MétodoSe ejecuta cuando...
OnTriggerEnter2DDos triggers entran en contacto
OnTriggerStay2DPermanecen en contacto (cada frame)
OnTriggerExit2DSe separan

Etapa 4 — Gráficos y Animaciones

Haciendo que tu juego vea bien: sprites, animaciones, efectos

🖼️ Sprites y Texturas

Un sprite es una imagen 2D que representa un objeto del juego. En Unity, importas imágenes (PNG, JPG) y las configuraste como sprites.

Contenido pending: cómo importar sprites, configurar el Sprite Renderer, atlas de texturas...

🎬 Sistema de Animaciones

Para animar un personaje, necesitas:

  • Una secuencia de sprites (fotogramas)
  • Un Animator que controle las transiciones
  • Un Animator Controller que define los estados y condiciones
💡 Ejemplo: Si tienes 8 imágenes del personaje corriendo, Unity puede cambiar entre ellas automáticamente a cierta velocidad para crear la animación de carrera.

Etapa 5 — Publicació i Distribució

Compartir tu juego: compilación, empaquetamiento, distribuición

📦 Build Settings

Antes de exportar tu juego, necesitas configurar File > Build Settings. Aquí decides:

  • Plataforma de destino (PC Windows, Mac, Linux, Android, iOS, WebGL...)
  • Resolución de pantalla
  • Icono del juego
  • Splash screen (pantalla inicial con el logo de Unity)
✅ Passos básicos: 1) Build Settings, 2) Elige plataforma, 3) Click en Build, 4) Elige donde guardar el .exe (o .apk para Android)
