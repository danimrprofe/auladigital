Què és Linux?

Un sistema operatiu lliure, de codi obert i gratuït que mou el món.

🖥️ Definició

Linux és un sistema operatiu (SO) de codi obert creat el 1991 per Linus Torvalds. Com Windows o macOS, gestiona el maquinari de l'ordinador i permet executar programes, però té tres característiques que el fan únic:

🆓 Lliure i gratuït

El codi font és accessible a tothom. Qualsevol pot llegir-lo, modificar-lo i distribuir-lo.

🔒 Segur i estable

Dissenyat per funcionar 24/7 sense reinicis. És el SO més usat en servidors del món.

🌍 Universal

Funciona en mòbils (Android), servidors, superordinadors, routers, cotxes i molt més.

🐧 La mascota de Linux és Tux, un pingüí. El nom "Linux" ve de combinar "Linus" (el creador) amb "Unix" (el sistema que el va inspirar).

🧩 El Nucli (Kernel)

Linux en sentit estricte és el kernel (nucli): el programa central que fa de pont entre el maquinari i els programes. Gestiona la memòria, els processos, els dispositius i el sistema de fitxers.

Kernel Linux

  • Gestiona la CPU i la memòria RAM
  • Controla dispositius (disc, xarxa, USB)
  • Permet que múltiples programes funcionin alhora
  • Base comuna de totes les distribucions

Sistema complet (distro)

  • Kernel Linux +
  • Eines GNU (ls, cp, bash...)
  • Gestor de paquets
  • Entorn gràfic (opcional)
  • Aplicacions d'usuari
💡 Tècnicament, el que installem és GNU/Linux: el kernel Linux + les eines GNU del projecte de Richard Stallman. Però popularment es diu simplement "Linux".

Per què és important Linux?

Per a un programador o tècnic informàtic, Linux no és opcional — és essencial.

💼 Importància professional

🌐 Servidors i núvol

Quasi tot el backend d'internet corre sobre Linux. Si treballes en desenvolupament web, DevOps o sistemes, treballaràs amb Linux cada dia.

🔒 Ciberseguretat

Les eines de seguretat més potents (Kali Linux, Metasploit, Wireshark) funcionen nativamente a Linux. És l'entorn estàndard per a ethical hacking i pentesting.

🤖 IA i Ciència de dades

Els clústers de GPUs per entrenar models d'IA corren tots sobre Linux. TensorFlow, PyTorch i Jupyter s'originen en entorns Linux.

💰 Sortida laboral

Conèixer Linux augmenta significativament el valor professional. La majoria d'ofertes de feina en IT mencionen Linux com a requisit.

🆓 Llibertat i filosofia del codi obert

Linux és el major projecte de col·laboració humana de la història. Milers de programadors de tot el món contribueixen al seu codi de forma voluntària o pagada per empreses com Google, Red Hat o IBM.

🌍 La llicència GPL (GNU General Public License) garanteix que Linux sempre serà lliure: si algú el modifica i el distribueix, ha de publicar el codi dels seus canvis. Ningú pot "tancar" Linux.
Gratuït per sempre Codi obert i auditable Sense telemetria ni espionatge Control total del sistema Sense llicències corporatives

Història de Linux

De Unix als anys 70 fins al nucli que mou el món avui.

🕰️ Línia del temps

1969

Unix — Bell Labs (AT&T)

Ken Thompson i Dennis Ritchie creen Unix als laboratoris Bell d'AT&T. El primer SO multitasca i multiusuari. Escrit en C, un llenguatge que ells mateixos van crear. Va influir en tots els SOs posteriors.

1983

Projecte GNU — Richard Stallman

Stallman llança el projecte GNU amb l'objectiu de crear un SO completament lliure compatible amb Unix. Crea eines fonamentals (gcc, bash, ls, cp...) però li falta el nucli.

1991

Linux 0.01 — Linus Torvalds

Linus Torvalds, estudiant de 21 anys de Hèlsinki (Finlàndia), publica el primer kernel Linux com a projecte personal. El famós missatge al fòrum: "Estic fent un SO lliure, sols com a hobby..."

1992

GNU + Linux = GNU/Linux

El kernel Linux es combina amb les eines GNU sota la llicència GPL. Neix el primer sistema GNU/Linux complet i funcional.

1993

Debian i Slackware — primeres distros

Apareixen les primeres distribucions que empaqueten Linux per a usuaris: Debian (base d'Ubuntu) i Slackware (la més antiga que encara existeix).

2004

Ubuntu — Linux per a tothom

Mark Shuttleworth funda Canonical i llança Ubuntu, basat en Debian, amb l'objectiu de fer Linux accessible per a usuaris sense coneixements tècnics. Revoluciona l'adopció de Linux al desktop.

2008

Android — Linux als mòbils

Google llança Android, basat en el kernel Linux. Avui és el SO mòbil més usat del món, amb més de 3.000 milions de dispositius actius.

Avui

Linux domina el món

El kernel Linux té més de 30 milions de línies de codi i més de 15.000 contribuïdors. Empreses com Google, Microsoft, Intel i Red Hat paguen enginyers per millorar-lo.

Distribucions de Linux

El kernel Linux és comú, però existeixen centenars de "sabors" adaptats a cada necessitat.

🤔 Què és una distribució?

Una distribució (distro) és un paquet complet que inclou el kernel Linux + eines de sistema + gestor de paquets + entorn gràfic + aplicacions preinstal·lades. Cada distribució té els seus objectius, filosofia i públic objectiu.

💡 Hi ha més de 600 distribucions actives. Totes comparteixen el mateix kernel però difereixen en l'entorn gràfic, el gestor de paquets, la configuració per defecte i el públic al qual van dirigides.

🪶 Per què Lubuntu a classe?

Lubuntu és una variant oficial d'Ubuntu que usa l'entorn gràfic LXQt, molt lleuger. És perfecte per a màquines virtuals perquè consumeix molt poca RAM i CPU, deixant més recursos per al sistema amfitrió.

✅ Avantatges per a la pràctica

  • Funciona bé amb 1-2 GB de RAM
  • Arrenca ràpid a la MV
  • Compatible amb totes les eines que usarem
  • Té terminal i entorn gràfic

📦 Gestor de paquets

Usa apt (Advanced Package Tool), el mateix que Ubuntu i Debian. Permet instal·lar, actualitzar i eliminar programes amb una sola comanda.

Estructura de carpetes de Linux

El sistema de fitxers de Linux segueix l'estàndard FHS (Filesystem Hierarchy Standard).

🌳 Tot part de l'arrel /

A Linux no hi ha unitats com C:\ o D:\ de Windows. Existeix un únic arbre de directoris que comença a l'arrel, representada per /. Tots els discos, USBs i particions es munten com a subdirectoris dins d'aquest arbre.

/— Arrel (root): inici de tot el sistema de fitxers
├── bin/— Comandes essencials (ls, cp, mv, cat...)
├── boot/— Fitxers d'arrencada (kernel, GRUB)
├── dev/— Dispositius hardware (discos, USB, terminals)
├── etc/— Fitxers de configuració del sistema i aplicacions
├── home/— Carpetes personals dels usuaris
│ └── alumne/— Carpeta personal de l'usuari "alumne"
├── lib/— Biblioteques del sistema (com DLLs de Windows)
├── media/— Punt de muntatge per a USB, CD-ROM...
├── mnt/— Muntatge temporal de sistemes de fitxers
├── opt/— Programes opcionals de tercers
├── proc/— Sistema de fitxers virtual: info de processos en curs
├── root/— Carpeta personal de l'usuari root (administrador)
├── sbin/— Comandes d'administració del sistema
├── srv/— Dades de serveis (web, FTP...)
├── tmp/— Fitxers temporals (s'esborren en reiniciar)
├── usr/— Programes i biblioteques d'usuari
│ ├── bin/— La majoria de programes instal·lats
│ └── share/— Dades compartides (documentació, icones)
└── var/— Dades variables (logs, bases de dades, correu)

🏠 La carpeta home

Cada usuari té la seva pròpia carpeta dins de /home. Per exemple, si l'usuari es diu alumne, la seva carpeta personal és /home/alumne.

El símbol ~ (titlla) és una drecera que apunta sempre a la teva carpeta personal: 

# ~ és equivalent a /home/alumne (si l'usuari és "alumne")
cd ~          # va a /home/alumne
cd ~/Documents # va a /home/alumne/Documents
💡 A diferència de Windows, a Linux els fitxers i carpetes que comencen per punt (.) estan ocults. Per exemple: .bashrc, .ssh. Es veuen amb ls -a.

🔤 Linux distingeix majúscules i minúscules

A Linux (a diferència de Windows), els noms de fitxers i carpetes diferencien majúscules de minúscules. Aquests tres fitxers són completament diferents: 

fitxer.txt    ← fitxer diferent de...
Fitxer.txt    ← ...diferent de...
FITXER.TXT    ← ...diferent d'aquest

Terminal i comandes bàsiques

La terminal és la eina més potent de Linux. Aprendre-la és fonamental.

💻 Què és la terminal?

La terminal (o consola) és una interfície de text on escrivim comandes que el sistema executa. A Lubuntu s'obre amb l'aplicació LXTerminal o amb la drecera Ctrl+Alt+T. 

alumne@lubuntu-vm:~$ ← prompte de comandes
  │         │        │ └── $ = usuari normal (# = root)
  │         └──────────── nom de l'equip
  └──────────────────────── nom de l'usuari
💡 La shell és el programa que interpreta les comandes. La shell per defecte a Linux és Bash (Bourne Again Shell).

📋 Comandes essencials

Comanda Descripció Exemple
pwd Mostra el directori actual (Print Working Directory) pwd → /home/alumne
ls Llista el contingut d'un directori ls -la
cd Canvia de directori cd /etc
mkdir Crea un directori nou mkdir proves
touch Crea un fitxer buit touch hola.txt
cat Mostra el contingut d'un fitxer cat hola.txt
echo Imprimeix text per pantalla echo "Hola món"
clear Neteja la terminal clear
man Manual d'una comanda man ls
history Historial de comandes executades history

🔧 Opcions i arguments

Les comandes poden rebre opcions (modificadors que canvien el comportament) i arguments (objectes sobre els quals actuen). 

ls -l -a /home
 │   │   │   └── argument: directori sobre el qual actua
 │   │   └────── opció -a: mostra fitxers ocults
 │   └────────── opció -l: format llarg (detallat)
 └────────────── comanda

# Les opcions es poden combinar:
ls -la /home   # equivalent a ls -l -a /home
💡 Dreceres útils: Tab autocompleta noms de fitxers i comandes. ↑/↓ navega l'historial. Ctrl+C atura la comanda en curs. Ctrl+L neteja la pantalla.

🔍 Comandes d'informació del sistema

uname -a       # informació del kernel i sistema
whoami         # nom de l'usuari actual
hostname       # nom de l'equip
date           # data i hora actual
df -h          # espai en disc (human-readable)
free -h        # memòria RAM disponible
top            # processos en execució (com el Gestor de Tasques)
ps aux         # llista de tots els processos
ip a           # adreces IP de les interfícies de xarxa

Navegació pel sistema de fitxers

Moure's pel sistema de directoris de Linux amb eficiència.

📍 Rutes absolutes i relatives

📌 Ruta absoluta

Comença sempre per /. Especifica el camí complet des de l'arrel, independentment d'on ets. 

cd /home/alumne/Documents
cat /etc/hostname

📍 Ruta relativa

Comença des del directori actual. Usa . (directori actual) i .. (directori pare). 

cd Documents       # entrar a Documents
cd ..              # pujar un nivell
cd ../../etc       # pujar 2 i entrar a etc

🧭 Comandes de navegació

pwd                   # on soc ara?
/home/alumne

cd /var/log           # anar a /var/log (ruta absoluta)
cd ~                  # anar a la carpeta personal
cd -                  # tornar al directori anterior
cd ..                 # pujar un nivell

ls                    # llistar contingut
ls -l                 # format llarg (permisos, mida, data)
ls -a                 # mostrar fitxers ocults (comencen per .)
ls -lh                # mides en KB/MB (human-readable)
ls -lt                # ordenar per data de modificació

tree                  # arbre visual de directoris (cal instal·lar)

🔎 Cercar fitxers

# find: cerca fitxers al sistema
find /home -name "*.txt"         # tots els .txt a /home
find . -name "fitxer.py"          # cercar al directori actual
find / -type d -name "config"    # cercar directoris anomenats config

# grep: cercar text dins fitxers
grep "hola" fitxer.txt            # cercar "hola" al fitxer
grep -r "error" /var/log          # cercar "error" a tots els logs
grep -i "hola" fitxer.txt         # sense distingir majúscules
grep -n "hola" fitxer.txt         # mostra el número de línia

Editor de text a la terminal

Editar fitxers directament des de la línia de comandes sense entorn gràfic.

📝 Per què editar a la terminal?

Quan treballem en un servidor remot o sense entorn gràfic, necessitem editar fitxers de configuració, scripts i codi directament des de la terminal. Els editors més comuns són nano (fàcil) i vim (potent però complex).

🟢 nano — l'editor senzill

nano és l'editor recomanat per a principiants. Les dreceres es mostren a la part inferior de la pantalla. 

# Obrir (o crear) un fitxer amb nano
nano fitxer.txt
nano /etc/hosts       # editar un fitxer del sistema (amb sudo)
Drecera Acció
Ctrl+O Guardar el fitxer (Write Out)
Ctrl+X Sortir de nano
Ctrl+K Tallar la línia actual
Ctrl+U Enganxar la línia tallada
Ctrl+W Cercar text
Ctrl+G Ajuda
Ctrl+C Mostrar la posició del cursor

🔵 vim — l'editor professional

vim (Vi IMproved) és extremadament potent però té una corba d'aprenentatge elevada. Funciona amb modes: no escrius directament com al nano.

Modes de vim

  • Normal (per defecte): navegar i executar comandes
  • Insert (i): escriure text
  • Visual (v): seleccionar text
  • Command (:): comandes especials

Comandes essencials

  • i → entrar al mode Insert
  • Esc → tornar al mode Normal
  • :w → guardar
  • :q → sortir
  • :wq → guardar i sortir
  • :q! → sortir sense guardar
💜 El meme famós: "Com surto de vim?" → prem Esc i escriu :q! + Enter. Milers de programadors s'han quedat atrapats a vim per primera vegada!

⚡ Redirecció: crear fitxers des de la terminal

Sense obrir cap editor, podem crear i escriure fitxers usant redireccions: 

# Crear un fitxer amb contingut (sobreescriu si existeix)
echo "Hola món" > fitxer.txt

# Afegir contingut al final (no sobreescriu)
echo "Segona línia" >> fitxer.txt

# Veure el contingut
cat fitxer.txt
Hola món
Segona línia

Fitxers i directoris

Crear, copiar, moure, renombrar i eliminar fitxers i carpetes.

📁 Crear fitxers i directoris

# Crear fitxers buits
touch fitxer.txt
touch a.txt b.txt c.txt    # crear múltiples alhora

# Crear directoris
mkdir nova_carpeta
mkdir -p carpeta/subcarpeta/subsubcarpeta  # crea tota la jerarquia

📋 Copiar fitxers i directoris

# Copiar fitxers
cp origen.txt desti.txt          # copia amb nou nom
cp fitxer.txt /tmp/             # copia a una altra carpeta
cp *.txt /tmp/                  # copia tots els .txt

# Copiar directoris (cal -r de "recursiu")
cp -r carpeta_origen/ carpeta_desti/

🚚 Moure i renombrar

A Linux, mv serveix tant per moure com per renombrar — és la mateixa operació. 

# Renombrar un fitxer
mv nom_antic.txt nom_nou.txt

# Moure a una altra carpeta
mv fitxer.txt /tmp/

# Moure i renombrar alhora
mv fitxer.txt /tmp/nou_nom.txt

# Moure múltiples fitxers
mv *.txt /tmp/

🗑️ Eliminar fitxers i directoris

# Eliminar fitxers
rm fitxer.txt
rm *.txt                  # eliminar tots els .txt

# Eliminar directoris buits
rmdir carpeta_buida/

# Eliminar directoris amb contingut
rm -r carpeta/             # recursiu
rm -rf carpeta/            # recursiu i forçat (sense demanar confirmació)
⚠️ rm -rf és irreversible. A Linux no hi ha paperera de reciclatge a la terminal. Un rm -rf / esborraria tot el sistema. Comprova sempre el que escrius abans de prémer Enter.

🔎 Visualitzar contingut de fitxers

cat fitxer.txt          # mostra tot el contingut
head fitxer.txt         # primeres 10 línies
head -n 5 fitxer.txt    # primeres 5 línies
tail fitxer.txt         # últimes 10 línies
tail -f /var/log/syslog # veure el log en temps real
less fitxer.txt         # paginat (q per sortir)
wc -l fitxer.txt        # comptar línies

Automatització i fluxos

Pipes, redireccions i encadenament de comandes per automatitzar tasques.

➡️ Redirecció de sortida

Per defecte, les comandes mostren el resultat per pantalla. Amb redireccions podem enviar-lo a un fitxer. 

# > sobreescriu el fitxer
ls -la > llista.txt         # guarda la llista al fitxer

# >> afegeix al final
date >> log.txt             # afegeix la data al final del log

# 2> redirigeix errors
ls fitxer_inexistent 2> errors.txt

# &> redirigeix sortida normal i errors
make &> build.log

🔗 Pipes (canonades) |

El pipe | connecta la sortida d'una comanda amb l'entrada d'una altra. Permet encadenar comandes per fer operacions complexes. 

# Comptar quants fitxers hi ha a /etc
ls /etc | wc -l

# Cercar "error" als logs i mostrar-ho paginat
cat /var/log/syslog | grep "error" | less

# Llistar processos i cercar-ne un concret
ps aux | grep "firefox"

# Els 5 fitxers més grans d'una carpeta
du -sh * | sort -rh | head -5
💡 El pipe és una de les idees més brillants de Unix: cada programa fa una sola cosa molt bé, i els pipes els connecten per resoldre tasques complexes.

⛓️ Encadenament de comandes

# && executa la segona SOLS si la primera té èxit
mkdir nova_carpeta && cd nova_carpeta
sudo apt update && sudo apt upgrade

# || executa la segona SOLS si la primera falla
ping -c1 google.com || echo "Sense connexió!"

# ; executa les dues sempre (independentment del resultat)
cd /tmp ; ls

⏰ Cron — programar tasques automàtiques

Cron és el servei de Linux per executar comandes o scripts de forma automàtica en un moment determinat (cada hora, cada dia, cada dilluns...). 

# Editar el crontab de l'usuari actual
crontab -e

# Format: minut hora dia_mes mes dia_setmana comanda
# ┌───── minut (0-59)
# │ ┌─── hora (0-23)
# │ │ ┌─ dia del mes (1-31)
# │ │ │ ┌ mes (1-12)
# │ │ │ │ ┌ dia setmana (0=diumenge)
0 2 * * * /home/alumne/backup.sh   # cada dia a les 2:00
*/5 * * * * date >> /tmp/log.txt  # cada 5 minuts
0 9 * * 1 sudo apt update        # cada dilluns a les 9:00
📜 Aquest tema NO ENTRA a l'examen, però és important per entendre l'automatització avançada.

Scripts de Shell (Bash)

Crear programes que automatitzen seqüències de comandes.

📜 Què és un script?

Un script de shell és un fitxer de text que conté una seqüència de comandes de Bash. En executar-lo, el sistema executa les comandes una per una, com si les escrivíssim a mà. Permeten automatitzar tasques repetitives. 

#!/bin/bash
# La primera línia (shebang) indica quin intèrpret usar

echo "Hola, soc un script!"
echo "La data actual és:"
date
echo "El directori actual és:"
pwd

▶️ Crear i executar un script

1

Crear el fitxer

nano script.sh
2

Donar permisos d'execució

chmod +x script.sh
3

Executar-lo

./script.sh       # amb ./
bash script.sh    # o amb bash explícit

🔤 Variables, condicionals i bucles

#!/bin/bash

# Variables
NOM="Anna"
echo "Hola, $NOM"

# Condicional
if [ $NOM == "Anna" ]; then
    echo "Ets l'Anna!"
else
    echo "No ets l'Anna"
fi

for i in 1 2 3 4 5; do
    echo "Número: $i"

L'usuari root

El superusuari de Linux: poders absoluts i responsabilitat total.

👑 Qui és root?

root és el superusuari del sistema. Té permisos absolutament il·limitats: pot llegir, escriure i esborrar qualsevol fitxer del sistema, gestionar tots els serveis, i realitzar qualsevol operació sense restriccions.

Usuari normal

  • UID (User ID): ≥ 1000
  • Prompte: $
  • Accés limitat al seu /home
  • No pot modificar fitxers del sistema

root

  • UID: 0 (sempre)
  • Prompte: #
  • Accés total a tot el sistema
  • Pot fer i desfer absolutament tot
⚠️ Treballar sempre com a root és molt perillós. Un error tipogràfic podria esborrar tot el sistema. La bona pràctica és treballar com a usuari normal i elevar permisos solo quan calgui.

🔑 sudo — elevació puntual de permisos

sudo (SuperUser DO) permet executar una comanda concreta amb permisos de root sense necessitat de ser root. Demana la teva contrasenya per confirmar. 

# Sense sudo: permís denegat
apt update
E: Could not open lock file - open (13: Permission denied)

# Amb sudo: funciona
sudo apt update
[sudo] password for alumne: 

# Executar una shell com a root (usar amb precaució!)
sudo -i           # shell root interactiva
sudo su           # canviar a l'usuari root
exit              # tornar a l'usuari normal
💡 Quan el prompte mostra # en lloc de $, ets root. Actua amb màxima precaució.

👥 Fitxer sudoers — qui pot usar sudo

No tots els usuaris poden usar sudo per defecte. El fitxer /etc/sudoers defineix qui té permís. A Ubuntu/Lubuntu, els usuaris del grup sudo poden usar-lo. 

# Afegir un usuari al grup sudo (com a root)
usermod -aG sudo nom_usuari

# Veure a quins grups pertanys
groups
alumne sudo adm cdrom...

📦 Què és un paquet?

A Linux, els programes es distribueixen en paquets (fitxers .deb a Debian/Ubuntu). El gestor de paquets s'encarrega d'instal·lar-los, resoldre dependències, actualitzar-los i eliminar-los automàticament.

Gestió d'usuaris

Crear, modificar i eliminar usuaris i grups al sistema Linux.

👤 Fitxers del sistema d'usuaris

Linux guarda la informació dels usuaris en fitxers de text al directori /etc:

Fitxer Contingut
/etc/passwd Llista d'usuaris: nom, UID, GID, home, shell
/etc/shadow Contrasenyes encriptades (sols root pot llegir-lo)
/etc/group Llista de grups i els seus membres 
# Veure els usuaris del sistema
cat /etc/passwd
root:x:0:0:root:/root:/bin/bash
alumne:x:1000:1000:Alumne:/home/alumne:/bin/bash
#nom:x:UID:GID:descripció:home:shell

➕ Crear i gestionar usuaris

# Crear un usuari nou (amb carpeta home)
sudo adduser nom_usuari      # interactiu, crea home
sudo useradd -m nom_usuari   # no interactiu

# Canviar la contrasenya d'un usuari
sudo passwd nom_usuari

# Eliminar un usuari
sudo deluser nom_usuari
sudo deluser --remove-home nom_usuari  # i esborra el /home

# Modificar un usuari
sudo usermod -aG sudo nom_usuari  # afegir al grup sudo
sudo usermod -s /bin/bash nom    # canviar shell

# Canviar d'usuari
su nom_usuari                     # canviar a un altre usuari
su -                              # canviar a root

# Crear un grup
sudo groupadd nom_grup

# Afegir un usuari a un grup
sudo usermod -aG nom_grup nom_usuari

# Veure els grups d'un usuari
groups nom_usuari

# Eliminar un grup
sudo groupdel nom_grup

# Veure tots els grups
cat /etc/group

Permisos de fitxers

El sistema de permisos rwx de Linux: qui pot llegir, escriure i executar.

🔐 El sistema de permisos rwx

Cada fitxer i directori de Linux té tres conjunts de permisos: per al propietari, per al grup i per a tots els altres. Cadascun pot tenir tres permisos: llegir, escriure i executar. 

ls -l fitxer.txt
-rwxr-xr-- 1 alumne alumne 1234 gen 15 10:30 fitxer.txt
 │││││││││ │ │       │
 │││││││││ │ │       └── grup propietari
 │││││││││ │ └────────── usuari propietari
 │││││││││ └──────────── nombre d'enllaços
 ││└──┘└──┘└── permisos del grup | altres
 │└──┘──────── permisos del propietari
 └──────────── tipus: - fitxer, d directori, l enllaç
r

read
Llegir

w

write
Escriure

x

execute
Executar

🔢 Notació octal (numèrica)

Els permisos es poden expressar com a nombres. Cada permís té un valor: r=4, w=2, x=1. Se sumen per a cada conjunt:

Número Permisos Significat
7 rwx Llegir + escriure + executar
6 rw- Llegir + escriure
5 r-x Llegir + executar
4 r-- Sols llegir
0 --- Cap permís 
# chmod: canviar permisos
chmod 755 script.sh    # propietari=rwx, grup=rx, altres=rx
chmod 644 fitxer.txt   # propietari=rw, grup=r, altres=r
chmod +x  script.sh    # afegir permís d'execució a tothom
chmod -r  privat.txt   # treure permisos de lectura
chmod u+x fitxer      # afegir execució sols al propietari (u)

# chown: canviar propietari
sudo chown alumne:alumne fitxer.txt    # usuari:grup
sudo chown -R alumne carpeta/          # recursiu

Accés remot amb SSH

Connectar-se a un servidor Linux des d'un altre ordinador de forma segura.

🔐 Què és SSH?

SSH (Secure Shell) és un protocol que permet accedir de forma segura a una màquina remota a través de la xarxa. Tota la comunicació va encriptada.

Usos principals

  • Administrar servidors remots
  • Transferir fitxers de forma segura
  • Tunel·litzar connexions
  • Accedir a la MV des del host

Components

  • Servidor SSH: openssh-server (a la MV Linux)
  • Client SSH: ssh comanda (a qualsevol Linux/Mac) o PuTTY (Windows)

⚙️ Configurar SSH a la MV Lubuntu

1

Instal·lar el servidor SSH

sudo apt install openssh-server
2

Verificar que el servei està actiu

sudo systemctl status ssh
● ssh.service - OpenBSD Secure Shell server
   Active: active (running)
3

Configurar la xarxa de la MV

A VirtualBox: Configuració → Xarxa → Adaptador en mode Adaptador pont (Bridged) per obtenir una IP de la xarxa real, o NAT amb reenvio de ports.

4

Saber la IP de la MV

ip a      # o ifconfig si tens net-tools

🖥️ Connectar-se per SSH

# Sintaxi bàsica
ssh usuari@ip_o_nom_maquina

# Exemple: connectar a la MV Lubuntu
ssh alumne@192.168.1.50

# Connectar per un port diferent al 22 (el port per defecte)
ssh -p 2222 alumne@192.168.1.50

# Quan es connecta per primera vegada, demana confirmar la clau:
The authenticity of host '192.168.1.50' can't be established.
Are you sure you want to continue connecting (yes/no)? yes

# Un cop connecto, el prompte canvia al de la màquina remota:
alumne@lubuntu-vm:~$   ← ara estem a la MV!
✅ Un cop connectat per SSH, treballes a la terminal de la màquina remota com si estiguessis davant seu físicament. Tot el que escrius s'executa a la màquina remota.

📁 Transferir fitxers amb SCP i SFTP

# SCP: Secure Copy — copiar fitxers per SSH
# Des del local a la remota:
scp fitxer.txt alumne@192.168.1.50:/home/alumne/

# Des de la remota al local:
scp alumne@192.168.1.50:/home/alumne/fitxer.txt .

# Carpeta sencera (-r recursiu):
scp -r carpeta/ alumne@192.168.1.50:/tmp/

Servidor web amb Apache/Nginx

Configurar Linux com a servidor web per servir pàgines HTML.

🌐 Què és un servidor web?

Un servidor web és un programa que escolta peticions HTTP (navegadors, aplicacions) i retorna el contingut sol·licitat: pàgines HTML, imatges, fitxers, etc.

🔴 Apache

El servidor web més antic i usat. Molt configurable, gran comunitat. Fitxer de configuració: /etc/apache2/

🟢 Nginx

Més modern i eficient per a moltes connexions simultànies. Usat per llocs d'alt tràfic. Fitxer de config: /etc/nginx/

⚙️ Instal·lar i configurar Apache

1

Instal·lar Apache

sudo apt update
sudo apt install apache2
2

Verificar que funciona

sudo systemctl status apache2
# Obrir el navegador i anar a: http://localhost
# Ha d'aparèixer la pàgina de benvinguda d'Apache
3

Directori arrel del web

Les pàgines web es serveixen des de /var/www/html/. El fitxer principal és index.html.

4

Publicar la nostra pàgina

sudo nano /var/www/html/index.html
# Escriure el nostre HTML i guardar

🔧 Gestionar el servei Apache

# Iniciar, aturar i reiniciar el servei
sudo systemctl start apache2
sudo systemctl stop apache2
sudo systemctl restart apache2
sudo systemctl reload apache2    # recarrega config sense aturar

# Activar l'inici automàtic en arrencar Linux
sudo systemctl enable apache2

# Veure l'estat
sudo systemctl status apache2
💡 systemctl és la comanda per gestionar serveis (daemons) a Linux modern. Funciona igual per a SSH, MySQL, Nginx i qualsevol altre servei.

📄 Servir la nostra pàgina web

# Crear una pàgina HTML pròpia
sudo nano /var/www/html/index.html
<!-- Contingut del fitxer index.html -->
<html>
  <head><title>El meu servidor Linux</title></head>
  <body>
    <h1>Servidor funcionant!</h1>
    <p>Lubuntu + Apache2 a VirtualBox.</p>
  </body>
</html>
# Donar permisos correctes al fitxer
sudo chown www-data:www-data /var/www/html/index.html
sudo chmod 644 /var/www/html/index.html

# Accedir des del navegador del host (si la MV té IP 192.168.1.50)
# http://192.168.1.50
✅ Si accedeixes a la IP de la MV des del navegador del teu ordinador real i veus la teva pàgina HTML, el servidor web està funcionant correctament!
