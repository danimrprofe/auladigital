Què és la virtualització?

Executar un ordinador complet dins d'un altre ordinador, de forma aïllada i independent.

💡 Concepte de màquina virtual

Una màquina virtual (MV) és un ordinador simulat per software. Funciona com un ordinador real —té el seu propi sistema operatiu, disc dur i xarxa— però existeix com un fitxer dins el teu ordinador físic.

🖥️ Màquina amfitriona (Host)

L'ordinador físic real. Executa VirtualBox i conté les màquines virtuals com fitxers al disc dur.

📦 Màquina virtual (Guest)

L'ordinador simulat que s'executa dins del host. Té el seu propi SO, RAM assignada i disc virtual.

💡 En lloc d'usar ordinadors reals per crear una xarxa, emularem tots els equips dins d'un sol ordinador físic. Estalvia maquinari i permet fer proves sense riscos.

⚙️ Per a què serveix VirtualBox?

VirtualBox és un software de virtualització gratuït creat per Oracle. Permet crear i gestionar màquines virtuals amb qualsevol sistema operatiu.

Gratuït i codi obert Windows, Linux, macOS Snapshots (punts de restauració) Xarxes virtuals internes Carpetes compartides

📀 Fitxers ISO: instal·lar un SO en una MV

Per instal·lar un sistema operatiu en una MV s'usa un fitxer ISO: una imatge del disc d'instal·lació que VirtualBox munta com si fos un CD/DVD físic.

1

Descarregar la ISO

Fitxer .iso que conté tots els arxius d'instal·lació del SO.

2

Muntar-la a la unitat òptica virtual

A VirtualBox: selecciona la MV → IDE secundari mestre → Unitat òptica → Seleccionar fitxer de disc.

3

Arrencar la MV des de la ISO

La MV arrenca des de la unitat òptica virtual i inicia el procés d'instal·lació.

Configuració de xarxa i adreces IP

Connectar les dues màquines virtuals en una xarxa interna i assignar IPs fixes.

🔌 Pas 1 — Xarxa interna a VirtualBox

Abans de configurar les IPs dins dels SOs, cal que ambdues MVs estiguin connectades a la mateixa xarxa interna a VirtualBox. Això simula un cable de xarxa entre elles.

1

Obrir configuració de cada MV

Selecciona la MV → Configuració → Xarxa.

2

Adaptador 1 → Xarxa Interna

Canviar el mode a Red Interna (Internal Network).

3

Posar el mateix nom a les dues MVs

Nom de la xarxa: RedXP2003 (ha de ser exactament igual a les dues màquines).

⚠️ Repetir aquests passos en ambdues màquines virtuals. Si el nom de la xarxa és diferent, no es veuran.

🗄️ Pas 2 — IP fixa al Servidor (Windows Server 2003)

El servidor necessita una IP fixa per ser localitzable sempre a la mateixa adreça.

Panel de Control → Conexiones de red → clic dret Conexión de área local → Propiedades → TCP/IP

Camp Valor
Adreça IP 192.168.1.1
Màscara de subxarxa 255.255.255.0
Porta d'enllaç (buit)
Servidor DNS preferit 192.168.1.1 (ell mateix)

💻 Pas 3 — IP manual al Client (Windows XP)

Panel de Control → Conexiones de red → clic dret Conexión de área local → Propiedades → TCP/IP

Camp Valor
Adreça IP 192.168.1.2
Màscara de subxarxa 255.255.255.0
Porta d'enllaç 192.168.1.1
Servidor DNS preferit 192.168.1.1
💡 El client apunta al servidor com a DNS perquè quan s'uneixi al domini, el servidor serà qui resolgui els noms del domini.

🏓 Pas 4 — Provar la connexió amb ping

Un cop configurades les IPs, verifiquem que les dues màquines es comuniquen usant la comanda ping.

Des de Windows XP

ping 192.168.1.1
# Ha de respondre el servidor

Des de Windows Server

ping 192.168.1.2
# Ha de respondre el client
✅ Si reps respostes a les dues bandes, les màquines estan correctament connectades. Si no respon, revisa la configuració de xarxa i el firewall.

Domini Windows i Active Directory

Configurar el servidor com a Controlador de Domini i afegir el client al domini.

🏛️ Què és un Domini de Windows?

Un domini és una xarxa d'ordinadors gestionada de forma centralitzada per un servidor. El servidor decideix qui pot entrar, a quins recursos pot accedir cada usuari i quines polítiques s'apliquen als equips.

Sense domini (Grup de treball)

  • Cada PC és independent
  • Usuaris i contrasenyes locals
  • Difícil de gestionar en molts PCs
  • No hi ha control centralitzat

Amb domini (Active Directory)

  • Gestió centralitzada al servidor
  • Un sol usuari per accedir a tots els PCs
  • Polítiques aplicades automàticament
  • Recursos compartits de forma controlada

🗄️ Instal·lar Active Directory al servidor

Active Directory (AD) és el servei de Windows Server que gestiona el domini: usuaris, grups, equips i polítiques.

1

Executar dcpromo

Executar → dcpromo → Enter. Inicia l'assistent de configuració del Controlador de Domini.

2

Seleccionar opcions

Controlador de domini per a un nou domini → Domini en un nou bosc.

3

Introduir el nom del domini

Nom: aulainformatica.local

4

Finalitzar i reiniciar

Configurar contrasenya segura per a l'administrador d'AD. Reiniciar el servidor.

💡 Després d'instal·lar AD, el servidor ja és el Controlador de Domini (DC) de aulainformatica.local.

💻 Unir el client Windows XP al domini

1

Verificar que el DNS apunta al servidor

El client ha de tenir com a DNS preferit la IP del servidor: 192.168.1.1.

2

Obrir propietats del sistema

Clic dret a Mi PC → Propietats → Nom de l'equip → Canviar.

3

Seleccionar Domini i introduir el nom

Seleccionar Dominio → escriure aulainformatica.local → Acceptar.

4

Introduir credencials i reiniciar

Introduir usuari i contrasenya de l'administrador del domini quan es demani. Reiniciar.

✅ Després de reiniciar, a la pantalla d'inici de sessió podrem triar iniciar sessió al domini aulainformatica.local.

Gestió d'Usuaris i Grups

Crear usuaris i grups al domini amb Active Directory (dsa.msc).

👤 Crear un usuari al domini

Tots els usuaris del domini es gestionen al servidor amb l'eina Usuarios y Equipos de Active Directory (dsa.msc).

1

Obrir dsa.msc

Ejecutar → dsa.msc → Enter.

2

Nou usuari

Dins del domini → clic dret a Usuarios → Nuevo → Usuario.

3

Emplenar les dades

Nom: Juan Pérez · Nom d'usuari: jperez
Contrasenya provisional: !1234567a

4

Marcar "ha de canviar la contrasenya"

Activar l'opció "El usuario debe cambiar la contraseña en el próximo inicio de sesión" → Finalitzar.

👥 Crear un grup i afegir usuaris

Els grups permeten assignar permisos a molts usuaris alhora. En lloc de donar permisos un a un, es donen al grup i tots els membres els hereten.

1

Crear el grup

Usuarios → Nuevo → Grupo → Nom: Alumnos.

2

Afegir membres

Doble clic al grup Alumnos → pestanya Miembros → Agregar.

3

Cercar i confirmar l'usuari

Escriure jperez → Comprobar nombres → Aceptar. Ara jperez pertany al grup Alumnos.

💡 Els grups faciliten l'administració: si afegim 50 alumnes al grup Alumnos, tots tindran accés als mateixos recursos automàticament.

Recursos compartits i Directives de Grup (GPO)

Compartir carpetes en xarxa i assignar-les automàticament als usuaris del domini.

📁 Crear una carpeta compartida al servidor

1

Crear la carpeta

Al servidor, crear la carpeta C:\Recursos_Alumnos.

2

Compartir-la

Clic dret → Propietats → Compartir → Activar "Compartir esta carpeta" → Nom: Alumnos.

3

Configurar permisos de seguretat

Pestanya Seguridad → Donar permisos de lectura/escriptura al grup Alumnos. Eliminar els permisos del grup Users (massa ampli).

🔗 Connectar la unitat de xarxa des del client (manual)

Des de Windows XP, connectar manualment la carpeta compartida com una unitat de xarxa:

Mi PC → Herramientas → Conectar a unidad de red → Ruta: \\Servidor\Alumnos → Lletra: Z:

  • L'usuari jperez pot iniciar sessió al domini des de Windows XP
  • L'usuari jperez té accés a la unitat de xarxa Alumnos
  • L'usuari pot crear i modificar fitxers a la carpeta compartida

⚙️ Directives de Grup (GPO): assignació automàtica

Una GPO (Group Policy Object) és una directiva que s'aplica automàticament als usuaris o equips del domini. Amb una GPO podem assignar la unitat Z: automàticament a cada inici de sessió, sense que l'usuari hagi de fer res.

1

Obrir l'editor de directives

Ejecutar → gpedit.msc, o bé des de dsa.msc → clic dret al domini → Propietats → Directiva de Grupo.

2

Crear una nova GPO

Clic dret al domini → Nova directiva de grup → Nom: Assignació Unitat Alumnos.

3

Editar la GPO

Clic dret → Editar → Configuració d'usuari → Preferències → Configuració de Windows → Assignacions d'unitat → Nou → Unitat de xarxa.

4

Configurar la unitat

Ubicació: \\Servidor\Alumnos · Lletra: Z: · Marcar "Reconectar".

5

Filtrar la GPO al grup Alumnos

Pestanya Àmbit → Seguretat → Agregar el grup Alumnos amb permisos de Llegir i Aplicar.

Aplicar i verificar:

# Al client Windows XP, actualitzar les directives:
gpupdate /force

# Verificar quines GPO s'apliquen:
gpresult /R
✅ Després de reiniciar i iniciar sessió amb un usuari del grup Alumnos, la unitat Z: apareixerà automàticament a Mi PC, sense cap acció manual.

Guest Additions i transferència de fitxers

Millorar la integració entre la MV i l'ordinador real, i mètodes per passar fitxers.

✨ Què són les Guest Additions?

Les Guest Additions (Adicions per a convidats) són un conjunt de controladors i eines que milloren la integració entre la màquina virtual i el sistema amfitrió.

Millor rendiment gràfic Portapapers compartit Arrossegar i soltar fitxers Carpetes compartides Millor compatibilitat ratolí/teclat

Instal·lació a Windows XP:

1

Inserir la imatge de les Guest Additions

Amb la MV encesa: Dispositius → Insertar imagen de CD de las Guest Additions.

2

Executar l'instal·lador

Obrir Mi PC → unitat de CD → executar VBoxWindowsAdditions.exe.

3

Reiniciar la MV

Reiniciar per aplicar els canvis.
